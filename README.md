# Ocean Companion

Ocean Companion 是一个面向 Android 与早期 HarmonyOS 平台的桌面伴随式 AI Agent 原型。它以悬浮球常驻桌面，通过无障碍上下文、文件文本、长期记忆和自定义人格，在合适时机显示主动弹幕，并支持长按展开半透明长时伴随面板进行对话。

## 已实现功能

- 悬浮球：支持拖动、单击、双击、长按展开伴随面板。
- 长时伴随面板：竖屏占下半屏，横屏占右半屏，占屏比例可调。
- 主动弹幕：进入特定应用或定时触发时显示独立顶部弹幕，自动淡出，不默认写入长对话。
- 弹幕导入对话：只有弹幕仍在显示时打开长时伴随面板，才会将这句话导入长对话。
- AI API 配置：支持 OpenAI 兼容接口，可自定义 Base URL、API Key 和模型名。
- 模型服务预设：可按 OpenAI、DeepSeek、阿里云百炼等兼容服务填写配置。
- 人格与用户名：支持自定义用户名称、AI 名称、人格提示词和多种人格预设。
- 自定义悬浮图标：支持文字图标，也支持选择图片作为智能体图标。
- 长期记忆：将重要对话和伴随内容保存到 Room 数据库。
- 屏幕上下文：无障碍服务会记录当前应用包名和可见文本，供主动发言与长对话使用。
- 文件上下文：可选择文件，将文件来源放入上下文队列。
- 主动触发应用：可通过应用名、包名或关键词匹配特定应用，进入后主动发言。
- 读屏诊断：识屏页面显示当前读取来源、包名、更新时间和文本长度。

## 当前限制

- 真正的 MediaProjection 截图 OCR 仍未完整落地；目前 OCR 服务主要保留授权与队列结构。
- 屏幕理解主要依赖无障碍服务能读取到的文本，不等同于完整视觉理解。
- UsageStats 前台应用兜底需要用户在系统中授予“使用情况访问权限”。
- 不同厂商系统对后台服务、悬浮窗、无障碍和使用情况访问权限的限制不同，可能需要手动放行。

## 权限说明

Ocean Companion 需要以下能力才能正常工作：

- 悬浮窗权限：显示悬浮球、主动弹幕和长时伴随面板。
- 无障碍服务：读取当前应用包名和可见文本。
- 使用情况访问权限：在无障碍上下文缺失时兜底判断前台应用。
- 通知权限：维持前台服务通知。
- 网络权限：调用用户配置的 AI API。

## 构建方式

使用 Android Studio 打开项目根目录，然后运行 `app` 模块即可。

也可以使用命令行构建：

```powershell
.\gradlew.bat assembleDebug
```

构建完成后，调试 APK 位于：

```text
app/build/outputs/apk/debug/app-debug.apk
```

## 使用流程

1. 安装应用并打开 Ocean Companion。
2. 授予悬浮窗权限，并启动悬浮服务。
3. 在系统设置中开启 Ocean Companion 无障碍服务。
4. 在设置页填写 AI API 的 Base URL、API Key 和模型名。
5. 配置用户名称、AI 名称、人格提示词、主动发言间隔和触发应用关键词。
6. 切换到匹配应用，等待顶部主动弹幕出现。
7. 长按悬浮球打开长时伴随面板，继续结合屏幕上下文进行对话。

## 项目结构

```text
app/src/main/java/com/projectocean/oceancompanion/
├── ai/          AI 服务适配、自定义 OpenAI 兼容客户端与 Prompt 构建
├── agent/       上下文分析、共享屏幕上下文、触发器与决策引擎
├── memory/      Room 长期记忆、会话历史、DataStore 配置
├── ocr/         MediaProjection 与 OCR 封装预留
├── service/     悬浮窗、伴随面板、主动弹幕、通知、无障碍服务
├── ui/          Compose 中文界面、配置页、记忆页与识屏诊断
└── utils/       平台辅助工具
```
